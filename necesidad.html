📋 Requisitos Funcionales - Módulo de Bodega e Inventario

Especificación técnica para evaluación de software de inventario - Absolute Nicaragua

⚠️ CONFIDENCIAL - USO INTERNO ÚNICAMENTE: Documento para evaluación interna de la gerencia. NO compartir con proveedores de software.

Propósito: Este documento describe las funcionalidades que debe tener un módulo de bodega/inventario para cubrir completamente las operaciones de nuestra empresa. Sirve como lista de verificación para evaluar si el software contable con módulo de inventario que están ofreciendo satisface nuestras necesidades reales.

Nota sobre otros módulos: Los módulos de Contabilidad, RRHH, Ventas y Logística se mencionan únicamente en lo que impacta DIRECTAMENTE el funcionamiento del inventario. Este documento NO evalúa sistemas contables, de nómina o CRM completos.

¿Cómo usar este documento? Para cada funcionalidad listada, marque si el software propuesto: ✅ (Lo tiene), ❌ (No lo tiene), ⚠️ (Parcialmente). Si hay ❌ en requisitos críticos de BODEGA, el software NO es adecuado para nuestras operaciones.

Crítico
Catálogo de Materiales Multi-Tipo
Gestión de materiales con clasificación por tipos: Electrónicos, Hierros/Estructurales, Consumibles, Herramientas. Cada tipo tiene campos específicos (voltaje, dimensiones, etc.). Soporte para contenedores (Burra, Canasta, Barril).
Dependencias: Ninguna (es base del sistema)
Crítico
📥 Ingreso desde Órdenes de Compra
Registrar entrada de materiales vinculados a órdenes de compra. Soporte para múltiples monedas (USD/C$), tipo de cambio configurable, número de factura, proveedor. Genera PDF de ingreso.
Dependencias: Catálogo de Materiales, Órdenes de Compra
Crítico
🛠️ Fabricación / Transformación
Proceso de manufactura donde se consumen materiales del inventario y se producen nuevos materiales. Requiere aprobación de contabilidad. Genera solicitud con responsable y fecha.
Dependencias: Catálogo, Recetas (opcional), Stock disponible
Crítico
📤 Despacho con Solicitudes
Gestión de salidas de materiales mediante solicitudes con correlativo secuencial. Estados: Pendiente, En Proceso, Completada, Rechazada. Genera acta de despacho en PDF. Soporte para devoluciones parciales.
Dependencias: Catálogo, Stock disponible, Eventos/Proyectos
Crítico
🔄 Devolución de Materiales
Registro de materiales devueltos desde eventos/proyectos. Vinculación con despacho original. Clasificación: Bueno, Dañado, Faltante. Genera PDF de devolución.
Dependencias: Despachos previos, Catálogo
Alta
🔧 Mantenimiento de Materiales
Registro de mantenimientos preventivos/correctivos a materiales. Historial por material. Costos asociados, responsable, fecha próxima mantención.
Dependencias: Catálogo de Materiales
Alta
📉 Bajas de Materiales
Dar de baja materiales por daño, obsolescencia o pérdida. Requiere justificación y aprobación. Registro en auditoría.
Dependencias: Catálogo, Stock
Media
💸 Ventas de Materiales
Despacho de materiales para ventas activas. Vinculación con módulo de ventas. Genera documento de salida.
Dependencias: Módulo de Ventas, Catálogo
Alta
🏢 Múltiples Bodegas / Ubicaciones
Soporte para bodegas principales y secundarias. Traslados entre bodegas. Stock separado por ubicación física.
Dependencias: Catálogo
Alta
📦 Gestión de Contenedores
Materiales pueden estar dentro de contenedores (Burra, Canasta, Barril). Trazabilidad material → contenedor → ubicación.
Dependencias: Catálogo, Multi-bodega
Crítico
📊 Stock en Tiempo Real
Cálculo automático de stock basado en movimientos (ingresos - salidas). Vistas por bodega, por ítem, por evento, por proveedor. Alertas de stock bajo.
Dependencias: Todos los movimientos registrados
Media
🪪 Generación de Etiquetas y Fichas
Impresión de etiquetas con código QR/código de barras para materiales. Fichas técnicas individuales y generación masiva.
Dependencias: Catálogo
Alta
🔒 Auditoría y Trazabilidad
Log de todas las operaciones: quién, cuándo, qué cambió. Tabla logs_auditoria con usuario, tabla, operación, datos nuevos.
Dependencias: Sistema de usuarios

📋 Recetas / Plantillas de Eventos
Plantillas predefinidas de materiales por tipo de evento (boda, conferencia, concierto). Recetas específicas por código de evento. Lista de materiales con cantidades estándar.
Dependencias: Catálogo de Materiales
Alta
🎯 Recetas Específicas por Código
Recetas personalizadas para eventos recurrentes. Se activan automáticamente cuando se usa cierto código de evento. Materiales con cantidades fijas.
Dependencias: Recetas base, Catálogo
Crítico
📝 Solicitudes de Logística
Solicitud formal de materiales para eventos. Estados: Pendiente Bodega, En Proceso, Completada, Rechazada. Prioridades (Normal, Alta, Urgente). Modos: Por receta o manual.
Dependencias: Recetas, Eventos, Stock
Alta
🚚 Seguimiento de Despachos
Dashboard de logística con estado de todas las solicitudes. Vista de pendientes, en proceso, completadas. Filtros por fecha, evento, encargado.
Dependencias: Solicitudes, Despachos
Crítico
🧮 Cálculo Automático desde Receta
Al seleccionar una receta, automáticamente calcula cantidades totales de materiales necesarios basado en número de personas/días del evento.
Dependencias: Recetas, Parámetros del evento

Módulo de Ventas

Solo lo que impacta el despacho de materiales

Nota: Este módulo solo incluye funcionalidades que impactan el despacho de materiales. No es un sistema CRM o de ventas completo.
Crítico
👥 Gestión de Clientes
CRUD completo de clientes: crear, modificar, listar. Datos: nombre, contacto, RUC, dirección, teléfono, email.
Dependencias: Ninguna
Crítico
📝 Cotizaciones / Prefactura / Factura
Crear cotizaciones con múltiples items. El mismo documento debe servir como prefactura o factura final. Vinculación con clientes. Generación de PDF. Posibilidad de convertir cotización en factura directamente.
Dependencias: Clientes, Catálogo, Tarifarios
Crítico
📊 Tarifarios Múltiples (A, B, C)
Sistema de tarifarios de servicios (A, B, C u otros). Cada cliente o cotización debe poder asociarse a un tarifario específico. Precios diferenciados por tipo de cliente o volumen. Cambios de tarifario según negociación.
Dependencias: Catálogo de Servicios
Alta
🔧 Servicios
Catálogo de servicios ofrecidos. Crear y modificar servicios. Precios, descripciones, códigos. Asociación con tarifarios.
Dependencias: Ninguna
Crítico
🏟️ Eventos / Proyectos
Registro de eventos con cliente, fechas, ubicación, tipo. Código único de evento. Vinculación con cotizaciones y despachos.
Dependencias: Clientes
Crítico
💰 Registro de Ventas y Facturación
Registro de ventas activas. Vinculación con despachos de materiales. Genera documentos fiscales. Historial de ventas por cliente, evento o período.
Dependencias: Clientes, Eventos, Bodega, Tarifarios
Crítico
📒 Control de Cartera y Cuentas por Cobrar
Gestión de cuentas por cobrar por cliente. Registro de pagos parciales o totales. Control de saldo pendiente, fechas de vencimiento. Reporte de cartera vencida. Posibilidad de aplicar pagos a facturas específicas.
Dependencias: Ventas, Clientes
Alta
📈 Reportes de Ventas
Reportes de ventas por cliente, evento, período, vendedor. Totales facturados, pendientes de cobro, estado de cartera. Exportación a Excel.
Dependencias: Ventas, Cartera

Módulo de Recursos Humanos

Solo lo que impacta el control de inventario y responsables

Nota: Este módulo solo incluye funcionalidades para: (1) saber quién recibe materiales, (2) quién realiza movimientos en bodega, (3) control de asistencia básico. No es un sistema completo de nómina o RRHH.
Crítico
👥 Gestión de Empleados
CRUD de empleados: datos personales, cargo, departamento, salario, fecha ingreso, estado (activo/inactivo). Código único de empleado.
Dependencias: Ninguna
Crítico
📆 Control de Asistencia
Registro de entradas y salidas. Verificación de asistencia por empleado y fecha. Cálculo de horas trabajadas, retardos, ausencias.
Dependencias: Empleados
Alta
🏖️ Gestión de Vacaciones
Solicitud de pago de vacaciones. Cálculo de días disponibles. Historial de vacaciones tomadas. Reporte de vacaciones por empleado.
Dependencias: Empleados, Asistencia
Crítico
🧾 Cálculo de Planillas
Generación de planillas por período. Cálculo de salario base, horas extras, deducciones (INSS, IR). Reporte imprimible.
Dependencias: Empleados, Asistencia, Vacaciones
Media
📇 Tarjetas de Identificación
Generación de tarjetas de empleado con foto, código, nombre, cargo. Formato imprimible.
Dependencias: Empleados
Alta
💰 Pagos a Personal Eventual
Registro de pagos a personal temporal por evento. Montos, fechas, conceptos. Reporte para contabilidad.
Dependencias: Empleados, Eventos
Media
💵 Control de Viáticos
Registro y control de viáticos entregados a empleados. Rendición de cuentas, comprobantes.
Dependencias: Empleados, Eventos

Módulo de Contabilidad

Solo lo que impacta el control de inventario y aprobaciones

NOTA: Este módulo NO es un sistema contable completo. NO incluye: libros diarios, balances, estados financieros, conciliaciones bancarias, impuestos, etc.

Lo que SÍ se necesita para bodega: Órdenes de compra (para ingresos), tipo de cambio (USD/C$), aprobaciones de movimientos de bodega, y reportes valorizados de inventario.
Crítico
📋 Órdenes de Compra
Generación de órdenes de compra a proveedores. Múltiples items, cantidades, precios. Moneda (USD/C$). Aprobación requerida.
Dependencias: Proveedores, Catálogo
Crítico
✅ Flujo de Aprobaciones
Sistema de aprobaciones para: fabricación, bajas, órdenes de compra, despachos grandes. Notificaciones a aprobadores.
Dependencias: Usuarios con roles
Crítico
💱 Tipo de Cambio Multi-moneda
Manejo de dólares y córdobas. Tipo de cambio configurable. Conversiones automáticas en reportes.
Dependencias: Ninguna
Alta
📊 Reportes Contables
Reportes de movimientos valorizados. Costos por evento. Inventarios valorizados. Exportación a Excel.
Dependencias: Todos los movimientos registrados

Características Transversales del Sistema

Funcionalidades que atraviesan todos los módulos

Crítico
🔐 Usuarios y Roles
Sistema de autenticación. Roles: Admin, Bodega, Logística, Ventas, RRHH, Contabilidad. Permisos por rol. Usuario puede tener múltiples roles.
Dependencias: Ninguna
Alta
📊 Exportación a Excel
Exportar datos de todos los módulos a Excel (.xlsx). Bodega, Ventas, RRHH, Logística. Formateado adecuado.

Crítico

Generar PDFs para: ingresos, despachos, devoluciones, fabricación, cotizaciones, planillas, tarjetas. html2pdf o similar.

Media
🔔 Notificaciones del Sistema
Notificaciones push del sistema operativo. Alertas de: aprobaciones pendientes, stock bajo, eventos próximos.
Dependencias: Eventos del sistema
Alta
📈 Dashboards y KPIs
Vistas gráficas con indicadores clave: stock por bodega, movimientos recientes, eventos activos, pendientes de aprobación.
Dependencias: Todos los módulos
Alta
🔍 Búsquedas y Filtros
Búsqueda por código, nombre, apodo en catálogos. Filtros por fecha, bodega, evento, estado. Búsqueda en tiempo real.
Dependencias: Índices de base de datos
Alta
📜 Historial y Reimpresión
Listado de documentos generados con opción a reimpresión. Ingresos recientes, despachos históricos, devoluciones.
Dependencias: Tablas de movimientos
Crítico
🗄️ Base de Datos PostgreSQL
PostgreSQL con Supabase. Tablas principales: catálogo (3 tipos), movimientos (3 tipos), empleados, eventos, recetas, solicitudes, usuarios. Vistas de stock actual. Triggers para actualizaciones automáticas.
Dependencias: Infraestructura cloud
Alta
💻 Aplicación de Escritorio
Electron con actualizaciones automáticas. Instalador Windows (.exe). Barra de título personalizada. Notificaciones nativas. Offline-first.
Dependencias: Base de datos remota

Cobertura Requerida en BODEGA

✅ El software es ADECUADO si:

  • Cubre ≥95% de los 11 requisitos críticos de bodega (catálogo multi-tipo, ingresos, despachos, devoluciones, fabricación, stock en tiempo real)
  • Soporta multi-moneda (USD/C$) y tipo de cambio configurable
  • Permite flujos de aprobaciones para movimientos de inventario
  • Los reportes de inventario son completos y exportables
  • Existe trazabilidad de todos los movimientos (quién, cuándo, qué)

❌ El software NO es ADECUADO si:

  • Faltan ≥2 requisitos críticos de BODEGA
  • No soporta contenedores o ubicaciones múltiples
  • No hay trazabilidad de movimientos
  • Requiere conexión a internet constante
  • El módulo de inventario es "básico" y no cubre flujos de fabricación, mantenimiento o bajas
