Registro de Usuarios

El proceso de registro permite crear nuevas cuentas de usuario en el sistema Absolute. Esta funcionalidad está restringida a administradores y usuarios autorizados para mantener la seguridad y control del acceso.

Descripción General

La pantalla de registro permite crear nuevos usuarios asignándoles credenciales únicas y roles específicos. Cada usuario debe tener un nombre completo, nombre de usuario único, contraseña segura y un rol que determine sus permisos en el sistema.

Control de Acceso

Solo usuarios con permisos administrativos pueden acceder a esta funcionalidad. El registro incluye validaciones para asegurar la integridad de los datos y la unicidad de los nombres de usuario.

Pasos para Registrar un Usuario

  • 1

    Acceder al Formulario

    Desde el panel administrativo, navega a la sección de gestión de usuarios y selecciona "Registrar Nuevo Usuario".

  • 2

    Ingresar Datos Personales

    Completa el campo "Nombre completo" con el nombre y apellido del nuevo usuario.

  • 3

    Crear Credenciales

    Asigna un nombre de usuario único y establece una contraseña segura siguiendo las políticas del sistema.

  • 4

    Asignar Rol

    Selecciona el rol apropiado según las responsabilidades del usuario (Bodega, Logística, Contabilidad, etc.).

  • 5

    Confirmar Registro

    Revisa la información y confirma el registro. El usuario podrá iniciar sesión inmediatamente con sus credenciales.

Roles del Sistema

Bodega

Gestiona inventarios, movimientos de materiales y control de stock en almacenes.

Logística

Maneja despachos, recepciones y coordinación de entregas.

Contabilidad

Administra finanzas, facturación y reportes contables.

RRHH

Gestiona personal, nóminas y recursos humanos.

Ventas

Maneja clientes, pedidos y procesos de venta.

Administrador

Acceso completo al sistema, configuración y gestión de usuarios.

Consideraciones Importantes

Políticas de Contraseña

Las contraseñas deben tener al menos 8 caracteres, incluir mayúsculas, minúsculas y números. Se recomienda cambiar la contraseña inicial al primer inicio de sesión.

Unicidad de Usuario

Cada nombre de usuario debe ser único en el sistema. Si hay conflictos, el sistema mostrará un mensaje de error.

Asignación de Roles

Los roles deben asignarse según las responsabilidades reales del usuario. Un rol incorrecto puede comprometer la seguridad del sistema.
